# Issue優先度分析ワークフロー

オープンIssueを分析し、次に着手すべきIssueを決定するワークフローです。

## 1. オープンIssue一覧を取得
```bash
gh issue list --state open --json number,title,labels,createdAt
```

## 2. Issue詳細を確認
```bash
gh issue view <issue番号>
```

## 3. 優先度の判断基準
以下の観点から優先度を評価:

| 基準 | 高優先度 | 低優先度 |
|------|----------|----------|
| 影響範囲 | ユーザー全体に影響 | 限定的な影響 |
| 依存関係 | 他のIssueがブロックされている | 独立している |
| 緊急性 | バグ・セキュリティ問題 | 改善・機能追加 |
| 作業量 | 小〜中規模 | 大規模 |

## 4. 依存関係の確認
- そのIssueを解決する前に必要な前提条件はあるか
- そのIssueを解決すると他のIssueが解決しやすくなるか

## 5. 選出結果をまとめる
選出したIssueについて以下を明記:
- Issue番号とタイトル
- 選出理由
- 推定作業時間
- 着手順序（複数選出した場合）
