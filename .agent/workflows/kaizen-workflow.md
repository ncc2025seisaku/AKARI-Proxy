# 改善ワークフロー（Kaizen）

ワークフロー自体を改善するためのメタワークフローです。

## 1. 現在のワークフローを確認
```bash
ls .agent/workflows/
```

## 2. 改善点の洗い出し
以下の観点からワークフローを評価:
- 冗長なステップはないか
- 自動化できる部分はないか
- 抜け漏れしやすいステップはないか
- 実際の作業とワークフローに乖離はないか

## 3. ワークフローを更新
改善点を反映してワークフローファイルを更新。

## 4. 変更をコミット
ワークフローの変更もバージョン管理する。
```bash
git add .agent/workflows/
git commit -m "docs: ワークフローを改善"
git push
```
