# 発表用負荷テストデータ（デモサーバー: 127.0.0.1, v2+Brotli応答, 非同期デフォルト）
- 実行コマンド: `uv run --python 3.11 python loadtest/disaster_suite.py --demo-server --no-event-log --host 127.0.0.1 --port 0`
- すべてローカルで実行。詳細ログ: `logs/disaster_suite_summary.json`, `logs/disaster_suite_history.jsonl`

## シナリオ別結果（成功/総リクエスト, timeout数, 平均応答時間）
- ベースライン: 80/80 成功, timeout 0, 平均0.005s
- 遅延3000ms+ロス20%: 164/200 成功, timeout 36, 平均0.160s
- ジッター350ms相当: 200/200 成功, timeout 0, 平均0.179s
- ロス25%: 161/220 成功, timeout 59, 平均0.003s
- バースト高並列: 763/800 成功, timeout 37, 平均0.012s
- 中負荷持続: 400/400 成功, timeout 0, 平均0.041s
- 断続ドロップ: 117/150 成功, timeout 33, 平均0.002s
- 強フラップ: 200/200 成功, timeout 0, 平均0.082s
- フラップ緩和（再送強化）: 200/200 成功, timeout 0, 平均0.068s
- MTU揺れ近似: 225/240 成功, timeout 15, 平均0.026s
- 巨大レスポンス10MB: Brotli圧縮+チャンクで 4/6 成功, timeout 2, 平均0.277s
- 3000本相当ロード: 2954/3000 成功, timeout 46, 平均0.047s

## ひとこと
- socket timeoutを総時間で見るよう修正し、巨大レスポンスのタイムアウト緩和＆再送余裕を付与。4/6まで改善。
- フラップ系はハートビート/再送/NACK上限を緩めて 100% 完走。断続ドロップ系はまだ timeout が残るが粘り強くなった。
- 高ロス・高遅延・MTU揺れは成功率やtimeoutが小幅に増減。必要ならtimeoutをもう一段伸ばす余地あり。
