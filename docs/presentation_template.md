# AKARI Proxy 発表資料ひな形

社内/カンファレンス等で AKARI Proxy を紹介する際のスライド骨子です。各スライドの bullet をベースに、実際のデモ構成や数値を追記してください。

## Slide 3: 背景と課題
- 災害/輻輳時に HTTP/TCP が不安定になる具体例
- 現行手段（VPN/HTTP 圧縮等）の限界
- 最低限の Web 取得ニーズ（ニュース/行政情報など）

## Slide 4: 解決したいこと
- UDP ベースの独自プロトコルによる堅牢化
- ローカル/外部プロキシ分離で安全性と柔軟性を両立
- PC/モバイル/将来アプリへの展開を意識した設計

## Slide 5: ソリューション概要
- 「ローカルプロキシ」「AKARI-UDP」「外部プロキシ」の役割
- 利用者視点フロー（ブラウザ設定 → 閲覧）
- コンポーネントの技術スタック（Rust + Python + Web UI）

## Slide 6: システム構成図
- ブラウザ → ローカルプロキシ → AKARI-UDP → 外部プロキシ → Web
- 認証/暗号（PSK + HMAC-SHA256）の位置付け
- フィルタリング/帯域制御ポイント

## Slide 7: AKARI-UDP プロトコル
- ヘッダ構造（magic, version, message_id, timestamp, size, flags）
- ペイロード分割とチャンク設計
- HMAC による改ざん検知・リプレイ対策
- Rust ライブラリ + pyo3 バインディングの関係

## Slide 8: ローカルプロキシ詳細
- HTTP リクエスト解析とモード切替（リバース/検索 UI）
- JS/CSS/画像フィルタリングによる帯域削減
- `akari_udp_py` 経由で UDP パケット生成

## Slide 9: 外部プロキシ詳細
- UDP 受信から HTTP/HTTPS 取得までの流れ
- タイムアウト/リトライ/エラー応答の方針
- レスポンス分割と再送設計

## Slide 10: Web プロキシ UI
- `py/akari/web_proxy/static/index.html` の挙動
- URL 直指定 vs 検索クエリ切替
- Proxy 設定パネル/LocalStorage 保存
- ロゴ差し替え手順（`logo.png`）

## Slide 11: デモシナリオ
- 1) ローカルプロキシ起動 (`akari.web_proxy.server`)
- 2) ブラウザプロキシ設定 → 通常サイト閲覧
- 3) `scripts/demo_udp.py` で正常/エラーケース再現
- 4) `akari.debug_tool` で datagram を解析
- 必要機材・準備コマンド一覧

## Slide 12: 技術的ハイライト
- Rust/Python モノレポ構成と `uv` ベース開発フロー
- pyo3 によるネイティブバインディングと再利用性
- Debug ツール群 (`debug_dump`, CLI) と開発体験
